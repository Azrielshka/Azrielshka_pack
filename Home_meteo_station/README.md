# 🌦 Домашняя метеостанция на Arduino

Простой проект для начинающих, который позволяет измерять температуру и влажность воздуха (датчик DHT11), влажность почвы (аналоговый датчик) и выводить данные на LCD 1602 дисплей. Переключение между экранами осуществляется кнопкой, а светодиодная индикация (OK/BAD) подсказывает, находятся ли параметры в комфортной зоне.
В папке представлены **две версии** скетча:
- `Home_meteo_station.ino` — базовая версия с подробными комментариями.
- `Home_meteo_station_pro.ino` — улучшенная (продвинутая) версия с более компактным и эффективным кодом.

## 🔧 Компоненты
- Arduino Uno (или совместимая плата)
- Датчик температуры и влажности DHT11 🌡️
- Аналоговый датчик влажности почвы (с двумя щупами) 🌱
- LCD 1602 дисплей (с I2C или без) 🖥️
- Кнопка тактовая 🔘
- Резистор 10 кОм (если кнопка без подтяжки, но в коде используется INPUT_PULLUP)
- Соединительные провода и макетная плата

## 🔌 Схема подключения
| Компонент           | Пин Arduino |
|---------------------|-------------|
| DHT11               | 9           |
| Датчик влажности почвы | A0        |
| Кнопка (одна нога)  | 7           |
| Кнопка (вторая нога)| GND         |
| LCD RS              | 11          |
| LCD E               | 12          |
| LCD D4              | 5           |
| LCD D5              | 4           |
| LCD D6              | 3           |
| LCD D7              | 2           |

**Примечание:**  
- Для DHT11 питание подключается к 5V и GND.  
- Датчик влажности почвы питается от 5V и GND, его сигнальный пин — A0.  
- Кнопка подключается одним контактом к пину 7, другим — к GND (используется внутренняя подтяжка `INPUT_PULLUP`).

## ⚙️ Принцип работы
1. Программа постоянно считывает:
   - температуру и влажность воздуха с DHT11,
   - значение влажности почвы с аналогового входа A0.
2. Полученные данные анализируются на соответствие «комфортным» диапазонам:
   - температура: 20…24 °C → **OK**, иначе **BAD**;
   - влажность воздуха: 33…55 % → **OK**, иначе **BAD**;
   - влажность почвы: 300…600 условных единиц АЦП → **OK**, иначе **BAD**.
3. На LCD дисплей выводится информация. Кнопка переключает два экрана:
   - **Экран 0**: температура и влажность воздуха с пометкой OK/BAD.
   - **Экран 1**: влажность почвы (сырые показания АЦП) и её статус OK/BAD.
4. При каждом нажатии кнопки экран очищается и отображается новый режим.

## 📁 Версии скетча

### Базовая версия
Файл: `Home_meteo_station.ino`  
- Подробные комментарии на русском языке, идеально для первого знакомства.
- Использует последовательные вызовы `lcd.print()` для вывода.
- Статусы OK/BAD задаются через блоки `if-else`.
- **Известная особенность**: в оригинале была ошибка — для почвы всегда выводилось `OK` (исправлено в pro-версии).

### Продвинутая версия (pro)
Файл: `Home_meteo_station_pro.ino`  
- Более компактный и профессиональный код.
- **Тернарный оператор** для присвоения статусов:
  ```cpp
  mod_t = (t >= 20 && t <= 24) ? "OK" : "BAD";
- Формирование строк перед выводом с помощью sprintf и dtostrf (для float). Это уменьшает количество вызовов lcd.print() и упрощает изменение формата.
- Исправлена ошибка с влажностью почвы — теперь статус вычисляется правильно.
- Все строковые буферы объявлены локально в loop(), что экономит память и делает код прозрачнее.
  
### Основные моменты кода
- Используются библиотеки `LiquidCrystal` и `TroykaDHT`.
- Кнопка обрабатывается с антидребезгом через сравнение предыдущего и текущего состояний.
- Анализ значений и присвоение статусов (`OK`/`BAD`) выполняется в цикле `loop()`.
- Для отладки предусмотрена функция `otladka()`, которая выводит данные в Serial Monitor.

## 🚀 Возможные улучшения
- Добавить датчик освещённости и выводить его показания на третьем экране.
- Использовать I2C версию LCD для экономии пинов.
- Реализовать звуковую или светодиодную индикацию выхода параметров за норму.
- Заменить DHT11 на более точный DHT22.
- Выводить значения в более наглядной форме (например, шкала влажности почвы в процентах).
